# 这是一个怎样的项目？
---
你是否遇到过，把整本课本丢给 AI，它却总是遗忘、胡言乱语？或者面对新的各种资料转换转半天，找不到一个合适的路径？

你是否听说过 **Skill** ，却不知道该如何调用和发挥它的威力？

你是否积累了许多好用的学习类**提示词**，却找不到地方与他人分享？

你又或是想要一个可以较为好用、沉浸的**AI学习专用平台**？

还是说，你想要一个能够快速导入课本告诉你各种信息的**应试助手**？

 **Learnova** 来了。这是一个**呢喃学生专属**(目前)的**Skill社区**与**极简AI对话式**平台。

**相比于**普通的大模型**对话网站**，你可以在这把课本转化为精准、轻量的**专属知识库** Skill ，用 Skill 设计各种独家的学习方式，或是导入、下载别人的学习资料作为知识库，导入、下载别人的提示词让AI以**更适合你的方式**输出。还可以体验独家的交互优化，让你跟AI的交互更省心。

当然，我们也在开发快速让大家把这些 Skill 导入到自己的agent助手中的方式（例如plugin market），我们注重的是这种学习方法本身能带给大家便利，而不是限于我们的**平台利用 Skill**。

通过对 Skill 的灵活应用，我们期望课本能以更轻量的方式载入个人的学习流程，让 Skill 作为一种新型的数字资料影响未来的学习模式的同时让AI更懂我们。






# More About Skills

---
**此外**，在这里，你也可以通过社区、通过与AI的交互快速地了解 Skill 等一些AI时代的新概念。也可以通过文档[[Skill科普]]快速了解智能体构建相关的知识和 Skill 的存在及其优势。

Agent设计的核心部分始终是**提示词**，各种各样的提示词带来LLM最终回答的质量。
 Skill 作为一种利用自然语言就可以调控AI行为的一种方法，能让几乎任何人都参与进来。
我们希望通过 Skill 能让大家跳脱出一些固定的功能，变成更**可定制的、更有自主性而且低成本**的形式。我们提供土壤，为大家介入AI更好地使用提供一个土壤。

除了对简单的 Skill 理解之外，我们也会输出一些*学习类* Skill **构建、迭代**的方法论，你也可以贡献自己的方法。但如上文，工具、路径肯定不止我们这一个，AI的回答形式、质量应该是多种多样的，**不应该受限于少数几个开发者所带来的思考上的停滞**，所以**社区**还是很有必要的。

无论如何，我们希望 Skill 这种纯文本的形式能够让大家都参与进 Skill 资源的创作中来。让 Skill 作为一种新型资源可以积累起来并进行共享。





# 除了社区本身，为什么采用对话作为交互的形式？

---
实际上，现在已经出现过很多教育类的agent/多agent系统，比如说[OpenMAIC](https://github.com/THU-MAIC/OpenMAIC)、[Khanmigo](https://www.khanmigo.ai/)还有很多实验性教育Agent或者教育辅助学习。

大部分都是比较注重**交互形式**，比如说各种roleplay，还有操作模式上新。交互形式固然是重要的，诚如“ 我们塑造了工具，此后工具又塑造我们 ”，交互形式上的方便可以在动机上和体验上会带来提升，我们也同样在进行前端的优化。但过于复杂的交互形式会带来额外的学习成本，甚至带来注意力的争夺。

所以最终采用了仅对话加其他拓展的形式。方便上手，而且拓展性高，挺多加进去的交互功能都是~~偷来的~~经过实践的。
